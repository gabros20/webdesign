The workflow arc

Six stages, distilled from a production website-rebuild pipeline. You can enter at any stage — "critique this page" jumps straight to 5–6; "build this from a design spec" starts at 4; a greenfield brand site runs 1→6.

direction → structure → craft → build → art review → critique

Stages 1, 3, 5, and 6 also stand alone — set a direction without building, critique a page you didn't build, or run the craft router for a single section.

directionDecide the taste/point-of-view — palette, type, register, the one signature — and record it as a formal design-system contract. → frontend-design-principles.md · design-direction.md
structureTurn the plan into a sitemap, navigation, footer, per-page section blueprints, and a deduplicated section catalog keyed to reusable archetypes. → section-archetypes.md
per-section craftFor each section, pick one bold move and the 1–2 devices that deliver it, then ship the CSS. Start from the craft router, load only the specific device references it points to (layout, typography, color, depth, decoration, imagery, motion, experimental, plus the strategy layer). → section-design-workflow.md
buildOne component per section key in a fail-loud registry, DESIGN.md tokens applied as a collision-free theme — shown in Next.js + Tailwind v4, portable to any stack. → frontend-build-patterns.md · tailwind-v4-theme.md
art reviewThe eye that set the direction judges the build section by section, at full resolution, and hands back an A0–A3 punch-list (or signs off). → art-review.md
critique / QA gateA scored, severity-routed pass — Nielsen heuristics, cognitive load, personas, mechanical checks — that turns "looks good" into a go/no-go, checked against the named AI tells. → design-critique.md · anti-default-catalog.md
An A0–A2 finding at review or critique sends the build back through the same stage until it's clean — no skipping, no downgrading a finding to a pass.

The non-negotiable laws

These govern every technique in the references — the load-bearing ones, condensed.

  • Depth over flatnessResolve every section into ≥2 planes, or at least one unifying texture/glow/shadow. A flat text-plus-image-square on a solid fill is boring.
  • Contrast creates interestManufacture a real jump in scale, weight, color, or depth. Exactly one element per section is unambiguously the biggest/boldest.
  • One bold move + restraintSpend boldness in ONE place, keep everything else quiet. Two bold moves compete and cancel; zero is generic.
  • Design for the businessSet the premium↔information dial before the moves — sequence a restrained hero into progressively denser proof, decided by stakes & emotion, not taste.
  • Register firstIs the design the product, or does it serve it? Brand/marketing takes the risk; dashboards/app shells earn familiarity instead. Name the register per surface.
  • Refuse the named defaultsModels default to the same look regardless of subject. Run every draft against the anti-default catalog — and avoid the predictable rebound too.
  • Real content, real assetsSpecific copy and real asset libraries over lorem, stock, emoji-as-icons, and hand-rolled SVG. Generic content is the loudest AI tell.

Start here

Install once. It's a plain SKILL.md router — no flags, no config, no scripts — so it activates on design/build/critique language ("design a landing page", "art-direct this hero", "critique this build") rather than a fixed command.

# skills.sh ecosystem npx skills add gabros20/webdesign-skill # clone + installer (per-host targets) git clone https://github.com/gabros20/webdesign-skill && cd webdesign-skill ./install.sh codex # or: claude | agents | cursor | antigravity | opencode | grok | hermes | all # use /webdesign art-review this build against its DESIGN.md — full-resolution, section by section /webdesign author a DESIGN.md for this brand — keep the logo, new layout and motion
