# Design: worker orientation & the toolbox — v1.2.0

> **Implemented in: v1.2.0** (2026-07-13, tag `v1.2.0`) — follows
> [v1.1.0-token-optimization.md](v1.1.0-token-optimization.md) · [CHANGELOG](../../CHANGELOG.md).
> Post-release audit fixes (brief-check report-path scoping, role-agnostic orientation wording,
> `scripts/check-sync`) shipped in **v1.2.1**.

## The gap (user-caught)

v1.1.0's token discipline protected briefs (priming anatomy) and cut worker output waste, but
never **licensed the worker's own orientation pass** — and three phrasings could be read as
forbidding it: "read targeted over whole files" (meant for tool-output hygiene), the
'"explore the codebase and…"' anti-pattern (meant for vague briefs), and "burns turns exploring"
(the failure is *blind* exploring, not exploring). A fresh worker MUST fill its context — tree,
manifests, conventions, neighboring patterns — or it produces convention-violating work.

## Options considered

- **A — instruction-only**: orientation license + graded ritual in the docs/blocks. No tooling;
  every worker re-probes what exists.
- **B — A + toolbox probe** *(chosen)*: `scripts/toolbox` probes the repo's tooling once (pure
  shell, zero tokens) into `.orchestrate/toolbox.md`; workers read recipes instead of probing.
- **C — B + outline helper + consented installs**: rejected — reverses v1.1.0's
  no-bundled-tooling decision; install prompts add friction; maintenance surface.

Key tooling findings: the harness's Glob/Grep/LSP + `git ls-files` are already the
token-efficient path; raw `tree-sitter` CLI is a token trap (S-expression dumps) — the practical
tree-sitter tool is `ast-grep`; `tree` is not preinstalled on macOS, so recipes must degrade to
`git ls-files`. Probing cadence: probe is free (shell), so run at every workspace creation if the
file is absent; the caching is about not re-*reading*/re-probing with paid tool calls.

## What shipped

1. `scripts/toolbox [--refresh]` → `.orchestrate/toolbox.md` (structure/outline/search/recency
   recipes chosen from what's installed, manifests, package manager, conventions files);
   generated by `scripts/workspace`; smoke-tested.
2. Orientation section in `token-economy.md`: license ("pointers are entry points, not fences"),
   the 5-step graded ritual (map → shape ≤2 → rules ≤3 → one neighboring pattern → symbols
   before bodies; budget ≈5 calls small / ≈15 complex), per-role grading (reviewers targeted;
   loop cached in contract; integrator git-state; verifier/advisor/triage provided-material).
3. WORKER block + orientation sentences ("Orient before editing… reading to understand is work, not waste; scale it to the task");
   REVIEWER block first line now licenses judging the diff in surrounding context, outline-first.
   New measured sizes: WORKER ≈285 tok, REVIEWER ≈139 (honest-numbers updated).
4. Misreadable phrasings fixed (anti-pattern scoped to vague briefs; "blind-searching").
5. Briefs gain one standard pointer: `read: .orchestrate/toolbox.md — orientation recipes`.
