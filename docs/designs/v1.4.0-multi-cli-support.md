# v1.4.0 вЂ” Coding-agent-agnostic orchestrate (multi-CLI support)

**Status: PLAN (draft вЂ” not yet implemented).**
Research base: eight parallel web-research reports, 2026-07-14, in `.orchestrate/raw/`
(`research-standard.md`, `-claude-code.md`, `-codex.md`, `-cursor.md`, `-antigravity.md`,
`-opencode.md`, `-grok.md`, `-hermes.md`) вЂ” every claim there is tagged
CONFIRMED-with-source or UNCERTAIN. This doc cites them as `[standard]`, `[codex]`, etc.

## 1. Goal & non-goals

**Goal:** one orchestrate skill pack that installs on and works well in the seven target
agents вЂ” Claude Code, OpenAI Codex CLI, Cursor CLI, Google Antigravity (CLI `agy` +
IDE), opencode, xAI Grok Build, Nous Hermes Agent вЂ” **without degrading the Claude Code
experience**, which stays the reference implementation.

**Non-goals:** per-host forks of the skill (single source only); porting the *site*;
supporting sunset Gemini CLI as a first-class target (covered as legacy notes only вЂ”
Google killed it for non-enterprise tiers 2026-06-18 `[antigravity]`); Windows-native
(non-WSL) script support.

## 2. What the research established вЂ” a three-layer portability model

**Layer 1 вЂ” file format: already portable.** Agent Skills (SKILL.md, agentskills.io,
Anthropic-published open standard Dec 18 2025) is adopted by every target: Codex, Cursor
(v2.4), opencode, Antigravity, Grok Build, Hermes all parse SKILL.md with progressive
disclosure `[standard]`. Our pack conforms today: `name` matches dir, description 605
chars (cap 1024), body 97 lines (guidance <500), references/scripts layout is exactly
the spec's. The only frontmatter caveat: spec-honored fields are
`name/description/license/compatibility/metadata/allowed-tools`; everything else must be
tolerated-unknown (Claude Code: confirmed silently ignored; opencode: confirmed ignored;
others: likely-but-unverified `[standard В§3]`).

**Layer 2 вЂ” installation: solved by the ecosystem.** `npx skills add gabros20/orchestrate`
(Vercel Labs, skills.sh) already discovers our `skills/orchestrate/SKILL.md` layout
(flat `skills/<name>/` pattern вЂ” verified against this repo) and maps installs onto 72+
agents' directories with `-a <agent>` / `-g` flags `[standard В§4a]`. The convergent
cross-agent path is `.agents/skills/` (project) + `~/.agents/skills/` (user); nearly every
target *also* reads `.claude/skills/` for back-compat (Cursor, opencode, Grok, Copilot,
Amp explicitly `[standard В§2]`).

**Layer 3 вЂ” semantics: this is the actual work.** The skill's *content* assumes Claude
Code primitives: the Agent/Task tool with per-dispatch model pinning, SendMessage/teams,
the Workflow tool, AskUserQuestion, `run_in_background`, EnterWorktree, Ralph stop-hooks,
`~/.claude/*` state paths. Other hosts have partial, differently-shaped equivalents.
Making the pack agnostic means abstracting these into named primitives with per-host
bindings and honest degradation ladders вЂ” not deleting Claude-specific quality.

## 4. Design decisions

**D1 вЂ” Single source, runtime host adaptation (no per-host builds).** One SKILL.md +
references tree, installed everywhere verbatim. A new shared reference defines host
detection + capability bindings; strategies reference abstract primitives. Rationale:
per-host builds would triple maintenance and drift (this repo already fights drift with
`check-sync`); the standard exists precisely so one artifact serves all hosts.

**D2 вЂ” Six named primitives, bound per host.** All strategy/shared docs get rewritten
against: `DISPATCH(task, model?, background?)` В· `PARALLEL(n)` В· `MESSAGE(agent)` В·
`ASK_USER(question, options?)` В· `WORKTREE(name)` В· `LOOP(stop-condition)`. A single new
file `references/shared/hosts.md` carries: (a) a detect-your-host procedure (probe which
native tools exist вЂ” Agent tool в‡’ Claude Code; `invoke_subagent` в‡’ Antigravity; `skill`
tool в‡’ opencode; `delegate_task` в‡’ Hermes; else check `$GEMINI_CLI` etc. вЂ” worded as
"look at your own toolset", zero tokens wasted), (b) the capability matrix above, (c) the
binding table per primitive, (d) the degradation ladder: **native primitive в†’ xcli
shell-out в†’ solo with a warning**. Controllers state the resolved host + bindings in
`.orchestrate/run.md` at kickoff (auditable, same pattern as dimensions today).

**D3 вЂ” Claude Code stays inline-first; other hosts one indirection away.** To avoid
degrading the reference experience, strategy files keep concrete Claude Code invocations
where they exist today, tagged as the *claude binding*, with one line pointing at
`hosts.md` for other hosts. Token cost for Claude users: ~1 pointer line per file. Token
cost for other hosts: one extra reference read (~1вЂ“2k tokens) вЂ” acceptable, matches the
pack's progressive-disclosure design.

**D4 вЂ” Strategy availability becomes an explicit, honest gate.** Triage gains a step 0:
"which strategies does this host support?"
- Portable everywhere (native or via xcli floor): `staged`, `parallel`*, `hierarchical`,
  `advisor`, `adversarial`, `xcli`, `loop`.
  (*opencode/Hermes parallel = xcli shell-out or external `opencode serve`/API driving.)
- `team` в†’ Claude Code (experimental teams) and Antigravity (`send_message`) only;
  elsewhere degrade to `hierarchical` with file-based handoff (already the pack's
  artifact-first doctrine, so degradation is mild).
- `workflow` в†’ Claude Code only; elsewhere degrade to a controller-scripted xcli loop.
Model-pinning rules (model-routing.md) gain a host caveat: on Antigravity/Grok/Hermes,
per-dispatch pinning is unavailable в†’ route via xcli engines when tier separation is
load-bearing (e.g. cheap-worker fleets), or accept single-model with effort knobs.

**D5 вЂ” xcli grows from 3 engines to 7 (this is a feature, not just parity).**
`xcli.md` currently covers codex/grok/claude. Add verified invocation blocks for
`cursor-agent -p` (+ACP note), `agy -p` (flag surface UNCERTAIN вЂ” must be verified against
a live install, per `[antigravity В§5]`), `opencode run`, `hermes -z`. Every engine block
keeps the existing rules (verify flags per session, `</dev/null` for codex, UUID sessions
for grok, no credential reading). This directly benefits Claude Code users too вЂ” more
peer/worker engines.

**D6 вЂ” Frontmatter: add spec fields, keep extensions tolerated.** Add `license: MIT` and
`metadata:` (repo URL, docs URL вЂ” also feeds skills.sh). Keep `version:` top-level (Hermes
wants it; everyone else ignores it). Do NOT add `allowed-tools` (experimental,
inconsistently honored). Body language changes: replace "invoke `/orchestrate`" with
host-neutral phrasing ("invoke the orchestrate skill вЂ” `/orchestrate` where slash
invocation exists"), since opencode invokes via its `skill` tool and Gemini-lineage hosts
gate activation behind consent `[standard В§3]`.

**D7 вЂ” Paths and hooks get host-conditional treatment.** `monitoring.md`, `evolve.md`,
`loop.md` references to `~/.claude/*` become a small per-host state-path table (Claude
`~/.claude/projects/`, Codex `$CODEX_HOME`, Grok `~/.grok/sessions`, opencode/Hermes/agy
equivalents). Ralph-loop section gains the Antigravity Stop-hook variant and Grok
`/goal`/`/loop` natives; external-driver loop (shell `while` around a headless CLI)
documented as the universal fallback. `isolation.md`: plain `git worktree add` commands
become the canonical text; EnterWorktree/`--worktree`/Grok-auto listed as host
accelerators.

**D8 вЂ” Distribution: three supported install paths, README-documented.**
1. `npx skills add gabros20/orchestrate` (recommended вЂ” handles all hosts' dirs; repo
   layout already compatible, verified `[standard В§4a]`).
2. `./install.sh [claude|codex|cursor|antigravity|opencode|grok|hermes|all]` вЂ” extend the
   existing script with the researched per-host paths (cheap, no Node needed). Keep
   symlink-vs-copy behavior consistent with npx skills' `--copy` default distinction.
3. Manual copy table in README.
README gets a "Supported agents" section with the capability matrix (condensed) and
per-host quirks (Hermes: explicit invocation only; Gemini-lineage: consent prompt on
activation; Cursor headless: no ask-user).

## 6. Quality safeguards (the "without degrading quality" contract)

1. **Claude Code is the regression baseline**: Phase-1/2 PRs must show a Claude-only
   diff review вЂ” any change that adds tokens or indirection to the Claude path needs a
   stated reason (D3 keeps it to pointer lines).
2. **check-sync grows two invariants**: hosts.md exists + every strategy file carries an
   availability line; SKILL.md line/token budgets enforced (honest-numbers pattern
   already in the script).
3. **No speculative flags**: every engine invocation block ships only commands verified
   in Phase 3 or carries the pack's existing "verify flags before trusting" rule
   inline вЂ” this rule already exists in xcli.md; it becomes load-bearing.
4. **Degradation is stated, never silent**: a controller on a host lacking a primitive
   must say so in run.md and to the user ("team unavailable on codex в†’ hierarchical").

## 7. Risks & open questions

- **Antigravity headless flags are unverified** (`--output-format`, approvals, CI auth вЂ”
  all UNCERTAIN `[antigravity В§5]`). Mitigation: Phase 3 live probe; until then agy is
  documented as "interactive-first, `-p` confirmed only."
- **Hermes has no confirmed project-level skills dir** and no auto-trigger вЂ” orchestrate
  must be invoked explicitly there; per-task model pinning is broken upstream (open
  issues). Hermes lands as a supported-with-caveats tier-2 host unless the user wants
  deeper investment (its Kanban board doc is an unexplored lead).
- **Cursor CLI custom commands are broken** (forum-reported) вЂ” we rely on skill
  invocation, not a `/orchestrate` command file; fine, but worth re-checking at Phase 3.
- **Spec governance**: Agent Skills is Anthropic-stewarded, not yet foundation-owned;
  field semantics could still move. Low risk вЂ” we stay on the required-fields core.
- **Open question for the user:** tier the seven hosts? Proposal: tier 1 = Claude Code,
  Codex, Cursor, opencode (full Phase-3 validation); tier 2 = Antigravity, Grok, Hermes
  (documented bindings, validated as installs allow).
