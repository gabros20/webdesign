# Design: raspberry accent + the per-skill color-identity system — v1.0.1

> **Implemented in: v1.0.1** (2026-07-15, tag `v1.0.1`) · [CHANGELOG](../../CHANGELOG.md).
> Site/presentation only — skill content and references unchanged (hence a patch release).

## The decision

Every published skill in this family (templated from the orchestrate repo) shares one design
system — same neutrals, type, layout, motion — and is differentiated by **exactly one accent hue**.
The hue *is* the identity: a growing skill family stays instantly recognizable per skill without
diverging design systems.

The mechanism: treat the **OKLCH hue wheel as the namespace**. Lightness and chroma are held at
the family constants (accent ≈ L 0.54 / C 0.185 light, L 0.69 / C 0.16 dark) so contrast math is
identical across every skill site; only the hue varies.

- **orchestrate = blue, hue ~265°** (`#3b63d8` light / `#6f93ff` dark) — taken first.
- **webdesign = raspberry, hue ~350°** (`#b6307d` light / `#e26aa8` dark) — this release. Chosen
  for the web-design context (warm, editorial, unmistakably not the default agent-tool blue) while
  respecting the skill's own anti-default catalog (not the terracotta cluster, not a
  purple→pink gradient identity — a single flat accent).
- Remaining well-separated slots for future skills: **gold ~80°**, **teal ~190°**, **green ~140°**.

## Where the accent must flow

An identity change is only real if it reaches every rendered surface. The v1.0.1 checklist, kept
here because every future reskin repeats it:

1. `site/index.html` CSS tokens (`--accent`, light + dark).
2. `remotion/src/theme.ts` — then **re-render** the hero animation videos and their poster frames
   (light + dark); a token change without a re-render ships a stale-colored video.
3. `site/og.png` (regenerated from the og source) and `site/favicon.svg`.

## Considered and rejected

- **Keeping the template blue** — rejected: blue is orchestrate's slot; two skills sharing a hue
  defeats the identity system.
- **A different design system per skill** — rejected: recognizability should cost one variable,
  not a redesign; shared neutrals also keep the template's contrast ratios and a11y intact.
- **Terracotta/warm-clay tones** — rejected by the skill's own anti-default catalog (the named
  "AI default" cluster).
