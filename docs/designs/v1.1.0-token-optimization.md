# Design: token & context optimization — v1.1.0

> **Implemented in: v1.1.0** (2026-07-13, tag `v1.1.0`) — grounded in
> [../research/token-optimization.md](../research/token-optimization.md) (opportunities O1–O14) ·
> [CHANGELOG](../../CHANGELOG.md). Twice reviewed by an external advisor (gpt-5.6-sol) — see
> *Advisor review* at the bottom.

## Goal & non-negotiable constraint

Cut token spend and context-window usage across every orchestrate strategy **without degrading
input quality**. The research's central number: token spend explains ~80% of multi-agent quality
variance — spending tokens *is* the capability. So the target is **zero wasted tokens, not minimal
tokens**: workers must be primed with complete, high-quality context; what gets cut is narration,
duplication, raw dumps, and unstructured returns.

## Design principles (derived from the research + advisor pass)

1. **Cut waste, not information.** Never trade brief completeness for size; underpriming costs
   more than overpriming (a worker missing context burns turns exploring, or round-trips
   NEEDS_CONTEXT — a full re-dispatch).
2. **Compress the middle, keep the boundaries dense.** Routine working narration → silence.
   Boundary artifacts (briefs, reports, reviews, escalations) → complete but structured; detail =
   information density, not length.
3. **Structural means executable.** "Be brief" regresses (documented); schemas and caps hold — but
   only if something *checks* them. This design ships validators (`scripts/brief-check`, ledger
   checks), not just wording.
4. **Prime with pointers, not payloads — pinned.** A brief is a map: exact paths (+ revision when
   it matters) with one line of why each. Inline content only when it's semantically complete,
   stable, and cheaper than rediscovery.
5. **Rules are role-scoped.** A 150-token contract on a 4-line verifier is net-negative (the
   caveman break-even lesson). Tool-heavy workers get the full contract; terse inline roles get
   one line; teams get explicit communication exemptions.
6. **The skill pays its own tax.** Every line added to SKILL.md or a template is paid on every
   run/dispatch — budgeted in the acceptance criteria.

## What changes (by file)

### 1. NEW `references/shared/token-economy.md` — the one reference for the token rules

Single on-demand file (~140 lines); strategy files name it, nothing is preloaded. Contents:

**a. The COMMS blocks — role-scoped, not universal** (advisor: one block does not fit all roles).

**WORKER block** (implementer, sub-orchestrator, integrator, loop executor — tool-heavy roles):

```
## Communication contract
Routine narration is silence: while working, don't announce tool calls, restate this brief,
or add pleasantries. Write text only when you find something load-bearing, change direction,
or hit a blocker. Required messages are NEVER silenced: task-state updates, approval requests,
integration/teammate coordination, security warnings, irreversible-action confirmations.
Minor choices — local, reversible, not user-visible, semantics-preserving (naming, formatting,
private helpers) — pick one and note it in your report. Defaults affecting security,
compatibility, persistence, or public behavior are NOT minor: resolve per your brief or
escalate.
Tool output: read targeted (grep, line-ranges) over whole files/logs. Redirect noisy commands
to a file at execution time (cmd > .orchestrate/raw/<task>-<what>.log 2>&1), inspect with
grep/tail; cite the minimum sufficient excerpt + the file path.
Blockers are structured, not brief: BLOCKED — what / evidence (excerpt + raw path) / what you
tried / what you need.
Reports: follow the schema, dense full sentences, state uncertainty and assumptions explicitly
— omit only rhetorical hedging, filler, arrow-chains, invented abbreviations.
When quoting literal code, commands, diffs, API names, or error strings: copy verbatim, never
paraphrase. Ordered multi-step instructions stay full prose.
```

**REVIEWER block** (spec/quality/panel/consensus roles): the WORKER block's tool-output +
verbatim + hedging lines, plus:

```
Report EVERY finding with severity + confidence — never self-filter to "important" ones;
triage is the controller's job. Findings go to the findings FILE (path in your brief); inline
return = verdict + counts by severity + file path.
```

**MINIMAL line** (advisor, triage assessor, verifier — short inline returns):

```
No preamble, no narration: return only your schema. Quote literals verbatim; state
uncertainty explicitly.
```

**Team strategy note:** teammates keep the WORKER block *minus* silence toward each other —
`strategies/team.md` exists because workers must talk; the exemption list above is restated there.

**b. Priming anatomy — "prime, don't dump"** (the input-quality half; O5 + /prime).
A quality brief contains, in order:

1. **Objective** — 1–2 sentences, the definition of done.
2. **Scope** — in / out, explicit. Include **negative constraints** (what must NOT change) and
   **rejected alternatives with reasons** — code can't show these; the brief must.
3. **Pointers, not payloads** — one per line, machine-checkable:
   `read: <path> — <why>` (optional suffix `@ <commit-sha>`; `(will exist)` marks generated
   artifacts brief-check must skip). Paths resolve from the repo root stated at the top of the
   brief. **Pin with `@ <sha>`** whenever more than one writer is active or the run uses
   worktrees — a branch name is not a pin.
   Inline content only when it is *semantically complete, stable, and cheaper than rediscovery*
   (an interface, invariants, a constraint table — verbatim); on high fan-out, inline the one
   canonical payload every worker needs rather than making N workers rediscover it.
4. **Interfaces & constraints to honor** — exact values, formats, relationships (verbatim).
5. **Verification** — the command(s)/check(s) that prove done.
6. **Report contract** — schema, cap, file path to write it.
7. **Stop conditions** — when to return BLOCKED / NEEDS_CONTEXT instead of guessing, and which
   decision classes REQUIRE escalation (security, data, public API).

**Brief probe-test (controller gate, pre-dispatch):** the brief must answer — *what exactly is
done? how is it verified? which files, at which revision? what must remain unchanged? can this
worker actually reach every pointer? what requires escalation?* Not answerable → fix the brief
now; a clarifying round-trip after dispatch costs a full worker turn.

Anti-patterns named: pasting session history; pasting whole files; "explore the codebase and…"
(unbounded reads); duplicating plan text a path reference covers; unpinned pointers in parallel
runs.

**c. Raw-output convention:** noisy commands are **redirected to file at execution time**
(`cmd > .orchestrate/raw/<task>-<what>.log 2>&1`), then inspected with grep/tail — the dump never
enters the transcript at all (saving after emission saves nothing; redirection is the input-side
win). Chat carries exit status + minimum sufficient excerpt + path. In worktree runs the store is
per-worktree; anything still needed after merge is cited (excerpt) in the task report or copied
next to it before cleanup — `parallel.md`'s integrator step gets that one line.

**d. Cache-friendly dispatch (best-effort hygiene, not a measured lever):** keep shared template
text stable across a run and put per-task variables late in the prompt; don't churn worker
model/tool configurations mid-run. (Claude Code manages provider caching internally; prefix
stability is hygiene we can control. Staggering identical-prefix fan-outs is documented as an
*option* only — unmeasured through the Agent-tool runtime, and the parallel strategy's batch
dispatch remains the default.)

**e. Controller diet:** the controller reads **reports by default, not diffs** — reviewers read
diffs via `scripts/review-package`. It retains **targeted inspection authority**: specific hunks,
symbols, or files when a gate is ambiguous, reviewers conflict, or a reviewer returns "cannot
verify from diff" (which review-gates already routes to the controller). Never paste file
contents into briefs; reference artifacts by path; ledger + `git log` on resume.

**f. Session hygiene (O7):** long orchestration runs deserve a lean controller session — unused
skills/MCPs cost ~1–1.5K input tokens per turn each; recommend `/checkup` (or a minimal-skill
profile) before big runs.

**g. Honest numbers (O8):** expect **10–25% session-level**, not 65–75% headlines; the only
trustworthy measure is an A/B on the provider's billing page. Multiplier context (≈4× single
agent, ≈15× multi-agent vs chat) so users first right-size *whether* to orchestrate (rule 7).

### 2. `references/shared/contracts.md` — grammar returns + findings files (O9, O11)

- Chat-return table gains a **line grammar** per role (cavecrew-derived), so terseness is
  structural: search/explore workers return `path:line — symbol — ≤10-word note` + totals;
  failure returns are single structured lines (`BLOCKED — what / evidence / tried / need`).
- **Reviewer returns change shape** (fixes a real contradiction the advisor caught): findings go
  to a FILE — collision-proof name `review-task<N>-<spec|quality|lens-<k>>-r<round>.md` — inline
  = verdict + counts by severity + path. Inline caps can no longer truncate findings.
  **Reviewer write scope clarified:** "read-only" means the REPO; `.orchestrate/` is the one
  place a reviewer writes (its findings file), stated in both reviewer templates.
- States the norm: **inline returns capped (≈1–2K max per role table); report FILES carry depth**
  and are read on demand.

### 3. `references/shared/review-gates.md` — the coverage rule (O10)

New paragraph under "Output review": **never instruct a reviewer to self-filter** ("only flag
high-severity", "be conservative", "at most N issues") — measured to silently drop recall.
Reviewers report **everything found, each with severity + confidence**; filtering/triage is the
controller's job at dedup/merge. Complements "never pre-judge findings". Terseness applies to a
reviewer's *prose*, never its *finding count*. Findings-file mechanics per contracts.md.

### 4. `references/shared/handoff.md` — probe-test (O13)

After writing a handoff (or any compaction summary), **probe it before trusting it**: can it
answer "which files changed, at which revision?", "what was the original error?", "what's the
next open item?", "what must not change?" from its own text + pointers? If not, regenerate —
artifact-trail loss is the measured worst failure mode of compression.

### 5. `references/prompts/*.md` — role-scoped blocks embedded

- WORKER block: implementer, sub-orchestrator, integrator, evolve (working passes).
- REVIEWER block: spec-reviewer, quality-reviewer (+ findings-file path parameter added to both
  templates' inputs and return sections).
- MINIMAL line: advisor, triage-assessor, verifier, planner-debate.
- No template loses an existing instruction; the implementer's "ask NOW if unclear" is
  *reconciled* with pick-and-note by the minor/not-minor definition (ambiguity in requirements,
  approach, dependencies stays ask-first; trivial local choices stay pick-and-note).

### 6. NEW `scripts/brief-check` — make "structural" executable

Small bash/awk validator run by the controller before dispatch:
`brief-check .orchestrate/task-N-brief.md` → verifies required sections present (objective,
scope, verification, report path, stop conditions), every `read:` pointer resolves on disk
(skipping `(will exist)`; warning on unpinned pointers when `git worktree list` shows >1
worktree), and the report path's directory exists. Exit non-zero, one-line reason per failure.
**Relationship to `scripts/task-brief`:** task-brief extracts the plan section; the controller
COMPOSES the brief (anatomy) around it — brief-check validates the composed brief, never raw plan
text, so existing plans stay valid. **Scope:** wired into the strategies that dispatch per-task
briefs (staged, parallel, hierarchical, team cards); loop/workflow/advisor/xcli have different
dispatch shapes — their contracts are the role blocks, exempt from brief-check by design (listed
in the coverage matrix, §11).

### 7. `SKILL.md` — one new universal rule (+ version bump)

File map gains `token-economy.md` + `brief-check`. Version → 1.1.0 (frontmatter + body line).

## Explicitly rejected (and why)

| Idea | Why not |
|---|---|
| A `verbosity=` dimension / caveman intensity knob | Silence-default for routine narration is always right; a knob adds surface + per-run decisions with no winning case. Default-on, role-scoped. |
| One universal COMMS block in all 10 templates | Advisor: net-negative on terse inline roles (~150 added input tokens vs <500-token returns); role-scoped variants instead. |
| Caveman styling for controller→user text | The user-facing boundary is where clarity has the highest leverage per token. |
| Full caveman skill as a dependency | ~1–1.5K/turn overhead, 0% input savings, net-negative on terse work — we take its carve-outs and measured anti-levers, not its persona. |
| Byte-identical-checksum acceptance criterion | Verifies duplication, not cache hits or savings (advisor). Kept as "shared core kept consistent" in review, not a gate. |
| Cache staggering as default | Unmeasured through the Agent-tool runtime; contradicts parallel strategy's batch dispatch; sacrifices latency. Documented as an option only (advisor). |
| Hard `<30 lines` inline-payload threshold | Arbitrary; replaced by semantic completeness + stability + rediscovery-cost rule (advisor). |
| Bundling RTK / context-mode / MCP proxies | External tooling with install friction; recommend in docs, stay dependency-free. |
| Task Budgets API | Beta, not exposed through the Agent-tool dispatch path today — watchlist note on `budget=`. |
| Compressing report FILES | Read on demand, carry the audit trail; compressing risks artifact-trail loss for near-zero savings. |

## Acceptance criteria (the skill pays its own tax)

1. Role-scoped blocks: each template embeds exactly its role's block. Budgets — MINIMAL ≤25:
   met (≈18). REVIEWER: the ≤60 estimate covered only the coverage add-on (≈51); the shipped
   self-contained block measures ≈115 (tailored reuse of the WORKER read/verbatim/hedging
   rules — the redirect line is deliberately absent for read-only roles). WORKER: design
   estimated ≤170; measured ≈244 —
   amended at implementation, kept intentionally: the overage is exactly the advisor-pass-1
   content (exemption list, minor-choice definition, structured blockers, redirect convention),
   and trimming it would reintroduce the misfires the advisor flagged. Recorded in
   token-economy.md's honest-numbers section.
2. SKILL.md grows ≤10 lines total.
3. No dispatch template loses an existing instruction (diff review: additive + semantics-preserving
   tightenings only); the implementer's clarify-first rule visibly reconciled with pick-and-note.
4. Coverage rule present in review-gates.md AND both reviewer templates; reviewer templates gain
   the findings-file path (brief input + return line).
5. `scripts/brief-check` exists, is executable, passes a smoke test (valid brief → 0; missing
   section / dead pointer → non-zero with reason), and is referenced by staged/parallel/
   hierarchical strategy files.
6. `.orchestrate/raw/` convention stated in token-economy.md and the WORKER block.
7. Honest accounting documented: per-dispatch block cost vs what it removes.

## Measurement plan (post-release)

Run **≥5 paired mid-size tasks** (5–8-task plans, staged strategy, same engine/models — v1.0.2
pins, unchanged across arms), each task once per version. Normalize token counts by diff size
(tokens per changed line). Track per run:

- input / output / cache-read tokens (provider usage page — the ground truth)
- **NEEDS_CONTEXT round-trips** and clarifying questions (underpriming signal — leading indicator)
- files read before first edit per worker; repeated reads of the same file (rediscovery signal)
- re-review rounds per task (quality signal); final review verdicts + any deferred-defect finds
- wall-clock per task

**Non-regression gate (predefined, directional):** keep v1.1.0 if median normalized tokens drop
≥10% AND NEEDS_CONTEXT round-trips don't rise by >1/run AND deferred defects (found after first
review) don't increase. Any quality clause fails → rollback is a PATCH loosening the WORKER
block, never the priming anatomy. If NEEDS_CONTEXT/rediscovery rises → the payload rule is
under-inlining; if re-review rounds rise → working-style is over-tight.

## Advisor review

**Pass 1 — 2026-07-13, `codex exec -m gpt-5.6-sol -c model_reasoning_effort=high --sandbox
read-only`, given the repo + research + design v1.** Verdict: direction right, five substantive
corrections — all adopted:

1. *"Structural guarantees" weren't executable* → added `scripts/brief-check` (§6) and the
   findings-file mechanics; acceptance criteria now test behavior, not wording.
2. *One COMMS block doesn't fit all roles* (break-even math; team strategy contradiction) →
   role-scoped WORKER/REVIEWER/MINIMAL blocks + team exemptions (§1a, §5).
3. *Wording that would misfire* — "defaults" wrongly minor; "no hedging" killing calibrated
   uncertainty; "shortest decisive line" losing causal context; report-file assumption on
   file-less roles; "byte-exact prose" self-contradiction → all reworded (§1a); raw output now
   saved to `.orchestrate/raw/` and cited by excerpt + path (§1c).
4. *Pointer briefs break down* on snapshot drift, inaccessible context, high fan-out, and
   negative constraints → pinned pointers, semantic inline rule, canonical-payload-on-fan-out,
   negative constraints + rejected alternatives required in scope (§1b); probe list extended.
5. *Controller-never-reads-diffs unsafe; measurement plan too weak* → targeted inspection
   authority retained (§1e); measurement redesigned around leading indicators (NEEDS_CONTEXT,
   rediscovery, re-review rounds) over single-run billing deltas.

Advisor cuts adopted: checksum criterion, default staggering, `<30 lines` threshold, model-family
update moved out (shipped as v1.0.2). Advisor additions adopted: brief-check, `/checkup` session
hygiene (§1f), raw-output store, instrumentation metrics. Partially adopted: tool-result clearing
(runtime doesn't expose it to the skill; noted in token-economy.md as a watchlist item alongside
Task Budgets).

**Pass 2 — 2026-07-13, same setup, on the revised design.** Verdict: **"ship with the following
amendments"** — all four adopted in this document: (1) brief-check validates controller-composed
briefs (not raw `task-brief` output) and its scope/exemptions are explicit + a dispatch coverage
matrix added (§6, §11); (2) machine-checkable pointer grammar with `@ <sha>` pinning and
`(will exist)` exceptions (§1b); (3) findings files made collision-proof and reviewer
read-only-ness scoped to the repo (§2); raw output redirected at execution time (not saved after
emission) with a durability rule for worktree cleanup (§1c); (4) measurement upgraded to ≥5
paired runs, normalized by diff size, with a predefined non-regression gate and rollback rule.
